# MOM智能问数-异步任务与重试设计

## 1. 目标

定义 Schema 初始化、知识向量化、导出、长查询等任务如何执行和重试。

## 2. 任务类型

- Schema 初始化
- 知识向量化
- 结果导出
- 长耗时查询
- ASR 代理超时重试

## 3. 分层选择

| 阶段 | 方案 |
| --- | --- |
| MVP 早期 | FastAPI `BackgroundTasks` 或同步任务 + 状态记录 |
| 任务变多后 | Celery / RQ + Redis |

## 4. 重试规则

1. 任务必须有唯一 `taskId`。
2. 任务必须能落最终状态。
3. 重试不能重复写业务结果。
4. 失败原因要可追踪。
5. Redis 只做短期状态，最终状态落 MySQL。

## 5. 与问数链路关系

- SSE 问数不依赖异步队列。
- 长任务单独排队，不阻塞主问数链路。
- 任务失败不能污染问数结果。
