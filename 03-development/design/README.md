# 设计文档入口

本目录存放智能问数 MVP 最终基准设计文档。后续开发任务必须以 `MOM智能问数-MVP开发Agent执行约束文档.md` 为最高基准。

## 推荐阅读顺序

1. `MOM智能问数-MVP开发Agent执行约束文档.md`：开发 Agent 的最高执行约束、禁止事项和任务边界。
2. `MOM智能问数-MVP全量文档终稿对齐报告.md`：终稿口径、MVP 边界、后置项和文档一致性结论。
3. `MOM智能问数-MVP接口设计文档.md`：后端 API、前后端契约、错误码和接口字段。
4. `MOM智能问数项目方案.md`：项目背景和阶段演进方向，仅作背景，不替代 MVP 实现清单。
5. `MOM智能问数-技术栈落地说明.md`：技术栈职责、引入时机和后续重栈边界。
6. `MOM智能问数-后端分层与代码结构.md`：后端分层和代码组织建议。
7. `MOM智能问数-Agent编排设计.md`：智能体工具调用、意图识别和结果拼装。
8. `MOM智能问数-RAG与向量库设计.md`：知识召回、向量库和 RAG 设计。
9. `MOM智能问数-异步任务与重试设计.md`：长任务、重试和队列策略。
10. `MOM智能问数-日志与链路追踪.md`：traceId、结构化日志和链路追踪。
11. `MOM智能问数-结果Payload扩展设计.md`：来源标识和推导过程扩展。
12. `MOM智能问数-ZHGC业务事实确认清单.md`：制造订单、质量、异常、SOP 的业务确认。

## 使用规则

- 开发任务入口先读执行约束文档，再读终稿对齐报告。
- 接口实现优先对齐 `MOM智能问数-MVP接口设计文档.md`。
- 数据库存储优先对齐 `docs/database/MOM智能问数-MVP数据库设计文档.md`。
- 需求范围优先对齐 `docs/requirements/` 下的最终用户侧和管理侧需求。
- 如果文档之间存在冲突，以执行约束文档和终稿对齐报告为准，并把冲突记录到 `findings.md`。
