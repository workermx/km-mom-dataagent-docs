# MOM智能问数-RAG与向量库设计

## 1. 目标

为 SOP、业务知识、术语解释、故障案例提供召回能力。

## 2. 适用范围

- SOP 操作指导文件
- 业务知识
- 智能体知识
- 同义词和字段口径解释
- 故障案例和维修记录摘要

## 3. 不适用范围

- 实时工单、库存、设备状态
- 语音报工报警
- OCR 智慧萃取平台
- 知识图谱和多跳推理

## 4. 推荐链路

1. 文档入库。
2. 切分。
3. embedding。
4. 向量写入。
5. TopK 召回。
6. 召回片段拼装为解释上下文。
7. 交给 Agent 生成自然语言解释。

## 5. 选型建议

| 能力 | 建议 |
| --- | --- |
| 向量存储 | PyMilvus |
| embedding | 独立模型服务或后端模型配置 |
| 召回结果 | 片段 + 来源标识 + 版本号 |
| 评测 | recall-test + 问句集 |

## 6. 设计约束

- RAG 不能覆盖实时事实查询。
- 召回结果必须保留来源引用。
- 召回失败时，文本问数仍应可用。
- 知识修改后向量状态应重建。
