# 后端落地检查表

> 支持件，不是主计划。只用于后端活动任务卡的自检。

## 门禁

- [ ] 是否已读取最高基准
- [ ] 是否已读取当前唯一活动后端计划
- [ ] 是否只做一个最小闭环
- [ ] 是否写清事实来源、涉及文件和验收命令
- [ ] 是否写清本轮不做范围

## 接口契约

- [ ] 请求字段是否和接口设计一致
- [ ] 响应是否统一为 `code/message/data`
- [ ] 是否使用 `agentId=default`
- [ ] 是否先保存消息，再通过 SSE 发起问数
- [ ] 幂等键是否为 `sessionId + userMessageId`
- [ ] 是否禁止旧 DataAgent 问数入口
- [ ] 是否禁止把废弃幂等字段作为正式后端口径
- [ ] 是否区分 `NO_DATA` 和错误

## 数据模型

- [ ] 会话模型是否包含 `sessionId`、`agentId`
- [ ] 消息模型是否包含 `messageId` 或 `userMessageId`
- [ ] Agent 回复是否能关联用户消息
- [ ] 结果 Payload 是否落到 `metadata_json.resultPayload`
- [ ] 是否保留 `traceId`
- [ ] 是否保留 `toolName` 和 `metricVersion`

## 编排规则

- [ ] 配置状态是否前置判断
- [ ] 权限是否由后端上下文注入
- [ ] 参数不足是否追问
- [ ] 未知问题是否阻断业务查询
- [ ] 工具调用是否只读受控
- [ ] 是否禁止 LLM 直接生成并执行 SQL
- [ ] 是否禁止调用未注册工具
- [ ] 是否做表范围和行数校验

## 测试

- [ ] 单元测试是否覆盖 Service 规则
- [ ] 接口测试是否覆盖确认实现字段
- [ ] 是否覆盖 `sessionId + userMessageId` 重试
- [ ] 是否覆盖 `NO_DATA`
- [ ] 是否覆盖异常场景
