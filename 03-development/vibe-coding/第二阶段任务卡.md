# MOM智能问数 第二阶段 VibeCoding 任务卡

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or executing-plans task-by-task. Each task card should be executed independently and verified before moving to the next one.

**Goal:** 把第二阶段需要补齐的业务事实、结果解释、权限隔离、导出报表、RAG 向量库、异步任务与链路追踪、评测回归，拆成可直接执行的 VibeCoding 任务卡。

**Architecture:** 先冻结业务口径，再补结果结构和治理能力，最后补知识检索与工程底座。所有任务都建立在 MVP 主链路已经可跑的前提上，`sessionId + userMessageId` 仍是幂等主键，不重做 MVP 已经约束过的部分。

**Tech Stack:** FastAPI、Pydantic、SQLAlchemy、Redis、pytest、PyMilvus、Celery/RQ、structlog、OpenTelemetry、openpyxl、reportlab

---

## 执行顺序

1. 任务卡 001：业务事实确认与口径冻结
2. 任务卡 002：结果 Payload 扩展与来源解释
3. 任务卡 003：权限与数据隔离最小闭环
4. 任务卡 004：导出与报表能力
5. 任务卡 005：RAG 与向量库底座
6. 任务卡 006：异步任务、重试与链路追踪
7. 任务卡 007：评测与回归体系
