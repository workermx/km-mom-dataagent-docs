# 提示词库

> 支持件，不是主计划。这里只放可直接复用的协作提示。

## 1. 只要方案

```text
先不要改代码。请结合当前文档和代码，给我一版最小可落地方案，列清楚会改哪些文件、为什么改、怎么验收。
```

## 2. 直接实现

```text
按当前活动计划直接实现这一个任务，保持最小修改，改完后跑测试并告诉我结果。
```

## 3. 只补文档

```text
先不动业务代码，把这一阶段的执行说明补成可直接落地的文档，重点是文件顺序、验收口径和常见坑。
```

## 4. 后端对齐

```text
开始做 Python + FastAPI 后端对齐，先把请求响应、幂等键、错误码、答案结构和会话模型定下来，不要先展开真实实现。
```
