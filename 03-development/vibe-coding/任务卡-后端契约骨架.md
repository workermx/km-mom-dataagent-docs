# 任务卡：后端契约骨架

## 目标

为智能问数 MVP 建立第一版 FastAPI 后端契约骨架，冻结统一响应、默认 Agent、会话、消息、SSE 占位和接口测试，为后续 MySQL、Redis、智能体编排、工具和 RAG 开发提供稳定入口。

- 接口设计：`docs/design/MOM智能问数-MVP接口设计文档.md`
- 数据库设计：`docs/database/MOM智能问数-MVP数据库设计文档.md`

## 范围

做：

- 建立 `backend/app` FastAPI 应用骨架。
- 建立统一响应结构 `code/message/data`。
- 建立默认 Agent 查询、会话创建/列表、消息保存/列表、SSE 搜索占位、停止占位接口。
- 建立 Pydantic Schema 和接口测试。
- 使用内存 Service 作为临时占位。

不做：

- 不连接真实 MySQL、Redis、LLM、RAG、ASR 或 MOM 业务库。
- 不生成或执行 SQL。
- 不实现真实权限中心、多用户、多租户或多 Agent 管理。
- 不把 DataAgent 的接口路径、响应结构或运行参数搬进本项目。

## 目标接口

- `GET /health`
- `GET /api/agent/default`
- `POST /api/agent/default/sessions`
- `GET /api/agent/default/sessions`
- `POST /api/sessions/{sessionId}/messages`
- `GET /api/sessions/{sessionId}/messages`
- `GET /api/stream/search`
- `POST /api/stream/stop`

## 契约约束

- 默认 Agent 固定为 `agentId=default`。
- 用户发送问题时先保存用户消息，再使用返回的 `userMessageId` 发起 SSE。
- 问数幂等键为 `sessionId + userMessageId`。
- SSE 问数入口为 `GET /api/stream/search?agentId=default&sessionId={sessionId}&threadId={sessionId}&userMessageId={userMessageId}`。
- `NO_DATA` 是成功空结果，不作为错误处理。
- 当前阶段服务可使用内存占位，但 README 必须明确后续接 Repository 和 MySQL/Redis。

## 验收

- OpenAPI 可访问并包含目标接口。
- 后端测试覆盖健康检查、默认 Agent、会话、消息、SSE 占位和停止占位。
- 测试断言 `userMessageId` 可用于消息回显和 SSE 占位。
- 代码中不新增旧 DataAgent 问数入口。
- 代码中不把废弃幂等字段作为后端正式问数口径。

## 验证命令

## 风险与回滚点

- 风险：当前后端仍是内存占位，不代表真实问数链路完成。
- 风险：数据库脚本仍需单独执行“口径修正”任务，不能把当前骨架视为数据库落地完成。
- 回滚点：删除本任务新增的 `backend/app` 和 `backend/tests` 文件，不影响现有前端 mock。
