# 迁移脚本目录

这个目录用于放后续增量迁移脚本。

## 规则

- `001_mvp_schema.sql` 是 MVP 基线，不要把后续增量混进基线脚本。
- 新迁移按顺序命名，例如：
  - `V202606250001__add_xxx.sql`
  - `V202606250002__alter_yyy.sql`
- 迁移脚本只做一件事，避免把建表、数据修正和清理混在一起。
- 破坏性迁移必须先写回滚说明和备份步骤。

## 当前状态

MVP 阶段先使用：

- `001_mvp_schema.sql`
- `002_mvp_seed_data.sql`
- `003_mvp_verify.sql`

后续有增量变更时，再把差异拆进这里。
