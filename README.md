# km-mom-dataagent-docs

智能问数文档仓库，用于按研发生命周期管理项目文档。

## 目录结构

| 目录 | 阶段 | 用途 |
| --- | --- | --- |
| `00-concept/` | 概念 | 沉淀问题背景、业务机会、目标用户、初步价值判断和术语定义。 |
| `01-planning/` | 规划 | 管理路线图、里程碑、范围边界、资源安排和阶段计划。 |
| `02-requirements/` | 需求 | 管理 PRD、用户故事、验收标准、交互说明和需求变更记录。 |
| `03-development/` | 研发 | 管理技术方案、接口设计、数据模型、实现说明和研发记录。 |
| `04-validation/` | 验证 | 管理测试计划、测试用例、验证报告、缺陷复盘和上线准入材料。 |
| `05-release/` | 发布 | 管理发布计划、版本说明、上线检查、运维交接和发布复盘。 |

## 使用约定

- 新文档优先放入对应阶段目录。
- 跨阶段材料可放在最早产生决策的阶段，并在后续阶段文档中引用。
- 长期有效的阶段规则或模板，可在对应阶段目录内再建 `templates/` 或 `guides/` 子目录。
